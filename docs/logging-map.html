The vocabulary (proposed names)

NameWhat it isIn the code today
OperatorThe human who launches and inspects runs.you
ConsoleThe web UI — the only thing the Operator clicks.agent-console/
Orchestrator / ControllerThe local app that runs everything: owns durable truth, spawns sessions, owns the push / PR / comments.run.py + agentloop/
Agent SessionOne conversation/run. Takes place inside exactly one Setting.a run / turn
Agent Session SettingThe place the session runs: Daytona sandbox, local Docker container, or bare metal. Swappable backend.sandbox.py, local_container.py, --local-direct
Agent ProgramThe program driving the conversation (pi). Emits the transcript.pi via OpenRouter
Workload programsEverything the agent runs: app build/serve, agent-browser, tests, lint.devcontainer serve, agent-browser

The four log sources

Log sourceEmitted byContainsFormLands in
1. Orchestrator — run logController The catch-all stream: controller's own logging plus echoed sandbox lifecycle and Workload stdout plus pi heartbeats. unstructured .agentloop-runs/run-*.log
2. Orchestrator — structured stateController Run snapshot, append-only transitions, per-iteration trace, assembled prompts, partial token usage. structured .agentloop-state/runs/<run>/, .agentloop-prompts/
3. Agent-program log (the conversation)Agent Program (pi) Thinking, input, output, tool calls & tool results — the actual reasoning trace. structured sessions/<agent>-<run>.jsonl (snapshotted to host)
4. Workload-program outputPrograms the agent runs Build/serve output, agent-browser output, test/lint stdout & stderr. unstructured no file of its own — echoed into the run log (source #1)

The thing worth talking about 👇

Notice sources 1 and 4 are the same file. There is no dedicated log for the Setting (the system hosting the session) and no dedicated log for Workload programs — they both blur into the one big run-*.log. Only the conversation (source 3) is cleanly separated. That's why troubleshooting means grepping across run-*.log + events.jsonl + trace.jsonl + the session JSONL + tmux logs. In v2 these four sources probably want to be four addressable streams attached to one Trace primitive per session.
